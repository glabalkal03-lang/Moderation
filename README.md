# Бот верификации (Discord) — No Name

Требования:
- Python 3.10+
- Discord бот-токен

## Установка

```bash
pip install -r requirements.txt
```

Скопируйте `config_example.py` в `config.py` и заполните значения:
- `TOKEN`
- `GUILD_ID`
- `UNVERIFY_ROLE_ID`, `MALE_ROLE_ID`, `FEMALE_ROLE_ID`
- `SUPPORT_ROLE_ID`
- `DENY_LOGS_CHANNEL_ID`, `REVIEWS_CHANNEL_ID`
- при необходимости `SERVER_DISPLAY_NAME`, `APPEAL_TEXT`

## Запуск

```bash
python bot.py
```

## Как это работает

- Саппорт вызывает `/action @участник`.
- Бот показывает кнопки: «Верефицировать» и «Недопуск» (ephemeral).
- При «Верефицировать» — выбор гендора (М/Ж), снимается роль `unverify`, выдается роль гендора, в ЛС отправляется сообщение с кнопкой «Оставить отзыв».
- При «Недопуск» — модальное окно причины, пользователю в ЛС приходит embed с причиной/кем/временем/апелляцией, а в канал логов отправляется запись «Недопуск ID: ...» с деталями.
- Кнопка «Оставить отзыв» открывает модал; текст отзыва публикуется в канале отзывов.

Важно: у бота должна быть роль выше, чем управляемые роли (unverify, гендерные) и права на отправку сообщений/управление ролями.

## Настройка текстов/ярлыков

Все подписи кнопок, заголовки модалок, тексты DM и шаблоны сообщений вынесены в `config.py` (см. примеры в `config_example.py`).

- Включение списка пресетов причин недопуска: `USE_PRESET_DENY_REASONS = True`
- Список пресетов: `PRESET_DENY_REASONS = ["..."]`
- Если пресеты выключены/пустые — при нажатии «Недопуск» откроется модал для ввода произвольной причины.
